用户面临的挑战

伴随着业务发展、UI 更迭，如何降低自动化脚本的编写和维护成本，在快速迭代中保障产品质量，是传统自动化面临的重要挑战。

准入门槛高
痛点描述
传统 UI 自动化框架需要编程基础，并且对测试框架有一定掌握，对测试人员技术能力要求较高。
方案描述
由于使用自然语言的方式编写测试脚本，测试人员无需具备编程能力和对测试框架的了解，上手易、门槛低。
脚本维护困难
痛点描述
团队内测试人员技术能力参差不齐，可能存在编码风格不统一，代码不规范等多种问题，导致多人协作维护难度大。
方案描述
多端统一的编写风格以及完备的语法支持，提高了脚本的可读性和可维护性，极大地降低了脚本的维护成本。
脚本兼容性不足
痛点描述
传统编写方式如 ID 与 Xpath 受框架、布局与机型等因素影响较大，测试脚本往往需要进行针对性编写，导致兼容性不足。
方案描述
自研的智能化技术脱离对页面结构的强依赖，在控件识别和定位上有效地弥补了传统测试框架的不足。
